Campo calculado (script)

Introducción​

Los campos calculados permiten la creación de fórmulas matemáticas que utilizan valores asociados a las respuestas para calcular diferentes valores, como promedios, valores máximos, valores mínimos, desviaciones estándar, etc.

Para crear una fórmula, puede utilizar cualquier tipo de campo/respuesta que tenga un valor numérico asignado.

Vea a continuación la lista de preguntas que tienen, o pueden tener, valores numéricos asociados y pueden usarse en una fórmula:

  • Preguntas numéricas (Enteros, decimales y cantidades usando Emojis)
  • Preguntas de Selección Única (Menú Desplegable, Radio y Botones)
  • Preguntas de selección múltiple (casillas de verificación y botones)
  • Problemas de fecha y hora
  • Preguntas de evaluación (Expresiones faciales y Emojis)
  • Problemas con la carga de archivos (cantidad de archivos cargados)

Creando una fórmula simple​

Imagina que quieres crear una fórmula que calcule el peso medio de una serie de muestras de un determinado producto. Entonces, imagina que el formulario tiene las siguientes preguntas:

  • Peso de muestra 1
  • Peso de muestra 2
  • Peso de la muestra 3

En este caso, cada pregunta debe configurarse con un nombre de campo único, por ejemplo, sample_weight_1, sample_weight_2 y sample_weight_3. // TODO: Agregar imágenes

Para hacer esto, debes crear un campo calculado y luego agregar la fórmula deseada:

function calcularValor() {
    return (resp.peso_amostra_1.number
          + resp.peso_amostra_2.number
          + resp.peso_amostra_3.number) / 3;
}

Analicemos cada parte de la fórmula:

  • función calcularValor() es la función que se ejecutará para calcular el valor del campo calculado. Esta función no se puede cambiar de nombre, siempre debe tener este nombre, no puede recibir parámetros y debe devolver un valor numérico o null (nulo/vacío).
  • resp es un objeto que contiene todas las respuestas del formulario.
  • resp.peso_amostra_1 es una estructura que contiene los datos asociados a la respuesta a la pregunta peso_amostra_1, como por ejemplo:
    • número es el valor numérico de la respuesta
    • name es el nombre del campo de la pregunta, por ejemplo peso_amostra_1
    • etiquetas es una lista de etiquetas asociadas con la pregunta
  • Finalmente, la fórmula calcula la media aritmética de los valores de respuesta, sumando sample_weight_1, sample_weight_2, sample_weight_3 y dividiendo por 3.
Nombres de variables

Puede utilizar cualquier nombre para las variables, pero se recomienda utilizar nombres que tengan sentido para su formulario.

Autocompletar

Puede utilizar las teclas Ctrl+Espacio o Ctrl+Shift+Espacio para activar el autocompletado manualmente.

Atributos generales de las respuestas.​

Todas las respuestas tienen al menos tres atributos: nombre, etiquetas y uno o más atributos específicos del tipo de respuesta, por ejemplo, número para preguntas numéricas, texto para preguntas de texto, etc.

nombre​

nombre: cadena

Nombre de la pregunta. Ejemplo: muestra_peso_1

etiquetas​

tags: string[];

Lista de etiquetas de preguntas, ejemplo: ['peso', 'muestra']

Las "etiquetas" se utilizan para agrupar preguntas y facilitar la creación de fórmulas. Por ejemplo, para calcular los pesos promedio de varias muestras, simplemente coloque una etiqueta en cada pregunta de peso, por ejemplo, #peso, y use esa etiqueta en la fórmula:

function calcularValue() {
    return avgByTag('#peso');
}

Donde avgByTag significa: "promedio por etiqueta", es decir, "promedio por etiqueta".

Atributos específicos de respuesta​

Vea a continuación la lista de atributos específicos disponibles para cada tipo de respuesta:

texto​

texto?: cadena;

Contenido textual de la respuesta, disponible para preguntas como:

  • Texto de una línea
  • Texto de varias líneas
  • Correo electrónico
Personaje "?"

El "?" indica que el atributo es opcional, es decir, puede existir o no. Si el usuario responde la pregunta el atributo existirá, de lo contrario no existirá.

etiqueta​

etiqueta?: cadena;

Etiqueta o Texto vinculado a la opción seleccionada, disponible para preguntas como:

  • Selección única
  • Selección múltiple
  • Expresiones faciales

número​

¿número?: número;

Contenido numérico de la respuesta, ya sea el contenido proporcionado por el usuario o el contenido numérico vinculado a la opción seleccionada. Disponible para preguntas como:

  • números enteros
  • Numeros decimales
  • Cantidad usando emojis
  • fórmula matemática
  • Selección única/simple
  • Expresiones faciales
  • Tiempo tiempo

fechaStr​

fechaStr?: cadena;

Fecha en forma de texto (cadena). Formato: AAAA-MM-DDTHH:MM:SSZ, por ejemplo: 2030-12-31T23:59:59Z

Disponible para preguntas como:

  • Fecha
  • Fecha y hora

latitud​

¿lat?: número;

Latitud de la coordenada geográfica reportada en la respuesta

lon​

lon?: número;

Longitud de la coordenada geográfica reportada en la respuesta

elementos​

¿elementos?: ISelectedItem[];

Lista de artículos seleccionados, disponibles para preguntas como:

  • Selección múltiple
  • Realizar pedidos

Cada elemento de la lista de "elementos" tiene los siguientes atributos:

  • label:string: Etiqueta o Texto vinculado a la opción seleccionada.
  • número?: número: Contenido numérico de la opción seleccionada.
  • posición?: número: Número que representa el orden elegido por el usuario para el artículo seleccionado. Disponible sólo en preguntas de tipo "Pedido"

Funciones disponibles​

Las siguientes funciones son herramientas que ayudan a crear fórmulas, permitiéndole realizar cálculos complejos en docenas o cientos de respuestas con solo una línea de código.

todo​

función todo(): IResponse[];

Devuelve una lista (array) con todas las respuestas existentes hasta el momento de ejecutar el código, es decir, las respuestas a las preguntas que aparecen antes de este elemento.

Atención

Esta advertencia se aplica a todas las funciones que devuelven una lista de respuestas.

buscarporetiqueta​

función findByTag(oneOrMoreTagNames: cadena | cadena[]): IResponse[];

Devuelve una lista (matriz) con respuestas que tienen las etiquetas especificadas.
Sólo se analizarán las preguntas que aparezcan antes de este elemento.

Ejemplo:

La siguiente función devuelve la cantidad de respuestas que tienen la etiqueta #weight, es decir, la cantidad de muestras completadas por el usuario.

function calcularValue() {
    const qntSamples = findByTag('#peso').filter(r => hasContent(r.number)).length;
    return qntSamples;
}

promedioPorEtiqueta​

función avgByTag(...oneOrMoreTagNames: cadena[]): número;

Calcula el promedio de los valores numéricos de las respuestas que tienen las etiquetas especificadas.

Las respuestas que no tengan valor numérico se ignoran.

Sólo se analizarán las preguntas que aparezcan antes de este elemento.

Consejo

Ej.: Si utiliza la siguiente fórmula para calcular el promedio de 3 pesos: (peso1 + peso2 + peso3) / 3, el resultado solo será correcto si se completan las 3 muestras. De lo contrario, el resultado será incorrecto.

Imagen que el usuario llenó solo 2 muestras, es decir peso1 y peso2, y dejó la tercera muestra en blanco, en este caso el resultado de la fórmula será: (peso1 + peso2 ) / 3 , que es diferente del resultado correcto: (peso1 + peso2) / 2.

Para evitar este tipo de situaciones se recomienda utilizar la función avgByTag para calcular el promedio, ya que maneja adecuadamente las respuestas en blanco, es decir, el resultado siempre será correcto.

sumaporetiqueta​

función sumaPorEtiqueta(...oneOrMoreTagNames: cadena[]): número;

Calcula la suma de los valores numéricos de las respuestas que tienen las etiquetas especificadas.

Sólo se analizarán las preguntas que aparezcan antes de este elemento.

contarporetiqueta​

función countByTag(...oneOrMoreTagNames: cadena[]): número;

Devuelve el número de respuestas que tienen las etiquetas especificadas.

Sólo se analizarán las preguntas que aparezcan antes de este elemento.

maxPorEtiqueta​

función maxByTag(...oneOrMoreTagNames: cadena[]): número | indefinido;

Devuelve el valor numérico más grande en respuestas que tienen las etiquetas especificadas o "indefinidas" si no se encuentra ningún valor.

Las respuestas que no tengan valor numérico se ignoran.

Sólo se analizarán las preguntas que aparezcan antes de este elemento.

minPorEtiqueta​

función minByTag(...oneOrMoreTagNames: cadena[]): número | indefinido;

Devuelve el valor numérico más pequeño en respuestas que tienen las etiquetas especificadas o "indefinidas" si no se encuentra ningún valor.

Las respuestas que no tengan valor numérico se ignoran.

Sólo se analizarán las preguntas que aparezcan antes de este elemento.

tiene contenido​

declarar función tiene contenido (valor: cualquiera): booleano;

Devuelve "verdadero" si el valor ingresado es diferente de "indefinido", "nulo", "[]" o una cadena vacía ('').

esta vacio​

declarar función está vacía (valor: cualquiera): booleano;

Devuelve verdadero si el valor ingresado es indefinido, nulo, [] o una cadena vacía ('').

Ejemplos​

Ejemplo 1​

En este ejemplo mostraremos cómo calcular los valores de la Fórmula Bhaskara. El propósito de este ejemplo es mostrar cómo almacenar valores parciales en variables y cómo usar estos valores para realizar cálculos más complejos.

En este ejemplo también veremos cómo utilizar funciones matemáticas predefinidas, como Math.sqrt y Math.pow.

Definición de la fórmula Bhaskara:

function calcularValor() {
    let a = resp.a.number;
    sea b = resp.b.número;
    sea c = resp.c.número;
    
    let delta = Math.pow(b, 2) - 4 * a * c;
    
    sea x1 = (-b + Math.sqrt(delta)) / (2 * a);
    sea x2 = (-b - Math.sqrt(delta)) / (2 * a);
    devolver x1; // o x2
}

Donde: Math.pow(b, 2) es equivalente a b²
Math.sqrt(delta) es equivalente a √delta

Ejemplo 2​

En este ejemplo veremos cómo calcular la secuencia de Fibonacci. O Objetivo deste exemplo é mostrar que é possível definir e usar funções diversas, inclusive funções recursivas, permitindo assim, a criação de cálculos mais complexos.

función calcularValor() {
    const n = resp.any_question_answer.number;
    return fibonacci(n);
}

función fibonacci(n: número): número {
    if (n <= 1) {
        return n;
    }
    devolver fibonacci(n - 1) + fibonacci(n - 2);
}
