# Como Criar ou Editar Formulários
Os formulários são a principal ferramenta para a realização da coleta de dados.
A criação correta de um formulário influencia diretamente no sucesso da análise dos dados coletados, pois garante as
informações relevantes sejam capturadas de maneira organizada.

A definição adequada das perguntas e dos tipos de resposta, como campos de números, de texto, data e hora ou múltipla escolha,
é fundamental para que os dados possam ser facilmente analisados e convertidos em conhecimentos para as tomadas de decisão.

Para criar ou editar um formulário, gerenciando suas funcionalidades, clique em <img src={useBaseUrl('img/start-here/main-menu/forms.webp')} alt="Formulário"
className="no-zoom image-between-texts" />, disponível no [Menu Principal](/start-here/main-page#main-menu).

## Organize os Formulários em Pastas {#organize-the-forms-into-folders}
Organizar formulários em pastas permite uma gestão mais eficiente, especialmente quando se lida com um grande volume de formulários.

As pastas e subpastas ajudam a categorizar os formulários por temas, projetos, datas ou departamentos, garantindo que cada um esteja no
local correto. Além disso, essa estruturação possibilita uma visualização rápida, facilitando a localização de formulários
antigos ou específicos, melhorando a colaboração entre equipes.

Quando um formulário for criado, será mostrada opção para que seja definida qual a pasta ou ou subpasta que o formulário
deverá ser criado.

### Gerenciamento de Pastas {#folders-management}
No gerenciamento de pastas, você pode:
* Incluir uma nova pasta;
* Incluir uma subpasta;
* Alterar o nome de uma pasta;
* Definir os grupos de colaboradores que terão acesso à uma determinada pasta, e;
* Excluir uma pasta.

## Adicione Elementos ao Formulário {#add-elements-to-the-form}
Após a criação do formulário, será aberta a tela de edição da estrutura e das funcionalidades formulário criado. Os **elementos**
são os componentes que constituem a **estrutura** de um formulário, permitindo a coleta e organização de dados.

:::danger Importante
As configurações realizadas nessa etapa definirão como o formulário será apresentado para a coleta de dados.
:::

Os elementos são compostos pelos seguintes campos:
* **Texto**:
* **Numéricos**: ;
* **Seleção Única**: ;
* **Múltipla Seleção**: Permitem a escolha de múltiplas opções de uma lista de opções;
* **Data e Hora**: facilitam a entrada de datas ou horas por meio de um calendário;
* **Avaliação**: permite pesquisa de satisfação por meio de expressões faciais ou emojis;
* **Exibição**: possibilita a exibição de imagens ou textos;
* **Outros**: como, enviar arquivo, GPS, campo calculado (*script* ou fórmula), ordenar/priorizar, fotos/imagens e assinatura.



### 1. Para adicionar elementos {#to-add-elements}
1. Clique em <img style={{width: '13%'}}  src={useBaseUrl('/img/forms/new-form/add-element.webp')} alt="Adicionar elemento"
className="no-zoom image-between-texts" />, disponível na porção inferior esquerda da tela;
2. Escolha o elemento desejado, observe que eles estão organizados em abas: **"Entrada de dados"** e **"Exibição"**;
3. Clique no elemento desejado e ele será incluído no formulário.

### 2. Para configurar os elementos {#to-setup-elements}
Os elementos adicionados ao formulário serão mostrados na porção central da tela e, ao redor desses elementos, serão
apresentadas as seguintes opções de configuração:

<table>
    <thead>
    <tr>
        <th style={{textAlign: 'start'}}>Opção</th>
        <th style={{textAlign: 'start'}}>Descrição</th>
    </tr>
    </thead>
    <tbody>
    <tr>
        <td>Texto principal do elemento</td>
        <td>
            O texto principal do elemento pode ser uma pergunta ou simplesmente um título. Por exemplo, para coletar o nome completo, pode-se usar:
            <ul>
                <li><strong>Nome Completo:</strong>, ou;</li>
                <li><strong>Qual o seu nome completo?</strong>.</li>
            </ul>
        </td>
    </tr>
    <tr>
        <td>Descrição</td>
        <td>
            Embora seja opcional, a descrição do elemento pode ser utilizada para instruir a resposta. Por exemplo, em um campo para coleta do nome completo, pode-se usar a seguinte descrição:
            <ul>
                <li><strong>Informe seu nome completo, sem abreviações</strong>.</li>
            </ul>
        </td>
    </tr>
    <tr>
        <td>Alterar o tipo da pergunta (elemento)</td>
        <td>
            Permite a alteração de um tipo do elemento para outros tipos relacionados, após ter sido incluído no formulário.<br />
            Por exemplo, altere um elemento do tipo <strong>Linha única</strong> para número <strong>Inteiro</strong>, ou um elemento <strong>Multi-seleção (Check)</strong> para <strong>Multi-seleção (Botões)</strong>.
        </td>
    </tr>
    <tr>
        <td>Ações condicionais</td>
        <td>
            São blocos do tipo "SE/ENTÃO": SE (condição acontecer) ENTÃO (faça a ação). <a href="#conditional-actions-learn-more">Saiba mais...</a>
        </td>
    </tr>
    <tr>
        <td>Conteúdo condicional (multiconteúdo)</td>
        <td>
            Criar vários conteúdos para a mesma pergunta, incluindo opções de respostas diversas. <a href="#conditional-content-learn-more">Saiba mais...</a>
        </td>
    </tr>
    <tr>
        <td>Opções de resposta</td>
        <td>
            Opção específica para alguns elementos que requerem configurações adicionais, como aqueles que permitem <strong>Seleção Única</strong> ou <strong>Múltipla Seleção</strong>.
        </td>
    </tr>
    <tr>
        <td>Clonar</td>
        <td>

<details id={"conditional-actions-lear-more"}>
    <summary>Saiba mais sobre **Ações Condicionais**</summary>

    Ações condicionais são aplicadas automaticamente nos elementos do formulário quando determinadas condições
    acontecem. São blocos do tipo **"SE/ENTÃO"**:
    * **SE** (condição acontecer) **ENTÃO** (faça a ação).

    Exemplo:

    Utilize as **ações condicionais** para ocultar ou exibir elementos do formulário quando determinada condição
    acontecer, tal como:
    * **SE* a opção X da questão Y for selecionada, **ENTÃO** oculte as questões A, B, C e exiba as questões D, E, F.


    > :warning: **Atenção**: As **ações condicionais** são processadas **SEMPRE** que uma resposta do formulário é
    alterada.

    [**Clique aqui**](#to-setup-elements) para retornar onde você estava.
</details>

<details id={"conditional-content-learn-more"}>
    <summary>Saiba mais sobre **Configurar Conteúdo Condicional (multiconteúdo)**</summary>

    É possível criar vários conteúdos para a mesma pergunta, incluindo opções de respostas diversas, escolhendo qual
    conteúdo deve ser exibido de acordo com condições configuráveis. Por exemplo, imagine duas perguntas:

    * Qual o Estado?
    * Qual a Cidade?

    Para facilitar a dinâmica ao responder o formulário, podemos exibir na listagem de cidades apenas as Cidades
    contidas no
    Estado selecionado. Para isso, basta habilitar o multiconteúdo (conteúdo condicional) na pergunta "Qual a Cidade?".

    Assim, é possível criar uma lista de cidades para cada Estado e vincular a exibição de cada lista com a seleção do
    Estado.

    A mesma lógica se aplica a vários outros cenários, por exemplo:

    - Setores, Áreas &larr &rarr ↔️ Departamentos
    - Departamentos ↔️ Pessoas, Gerentes
    - Categorias ↔️ Produtos, Itens etc.

    [**Clique aqui**](#to-setup-elements) para retornar onde você estava.
</details>
